First Experience with Linux Command Line:
Linux command line interface made me realize the power that command lines have even today. Their power lies in the simplicity. They are fast and more efficient than the GUI counterparts. The only difficulty is to get familiar with commands and various options associated with them.

Process Identifier (PID):

Every process within a Operating System will have a unique process identifier number assigned to them to uniquely identify them. Whenever user tries to terminate a process, all the child processes are terminated before terminating the parent process. In UNIX there are PID 0  and 1 which are reserved for the kernel.
  • swapper or sched is assigned PID 0 which is responsible for paging.
  • init process is assigned PID 1 which is responsible for starting and shutting down the system.
 
Pipe operation:
One of the interesting operations involves the use of Pipes. Pipes provide a dynamic way to concatenate operations. It redirects output from one operation to another operation.  
As an example to pipe operation lets try to find the PID of process from a list of processes by just specifying the name.


Running process of the user

Screenshot above shows the list of running processes on my system. Task is to find the Process ID(s) of process named 'cat'.
One of the approach is to use ‘grep’ command to print the pid.
ps | grep cat | cut -c1-5


First of the operation involves the use of ‘ps’ to find all the running process that current user has initiated. The output of ‘ps’ command is then feed as input to ‘grep’ command that tries to match a pattern ‘cat’ within the output. Finally ‘cut’ will try to cut exact a portion of the string. In this case it will try to skim characters 1 to 5 and display it on the screen.The result is as displayed below.
