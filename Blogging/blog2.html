Why I started liking GIT.....

I have been using Visual Studio IDE along with VSS and TFS as version control softwares for nearly two years before moving on to Git. To be frank, I was amazed how easily things could be controlled from a command line in Git. One of the cool features of Git is, it allows more than one person to edit the same file simultaneously.

GIT Philosophy:
“Commit more, Commit often”

Git workflow
 Git involves just 3 steps
  1. Get the latest copy of the repository on the local machine
  2. Make changes on the local copy and stage them
  3. Then commit the staged changes onto the server

  • Distributed Version Control
Git is distributed version control software. Being distributed means that all the participants will have an individual copy of the repository instead of repository residing on a central server.
  •  Simultaneous editing
More than one person can work on the same file.

  • Local Conflict resolution
Simultaneous edit could cause conflict. All conflict resolution takes place locally. Hence concerned persons are informed incase of conflicts. They need to resolve them locally before pushing them onto the master repository.

  • No Central Server
In git, server connection are required when one has to get new/update local copy and pushing changes to the master. This means one can work on projects even if there is no connection to the server.
  •  Cost
In comparision with VSS and TFS, git is cheap.

How to integrate Git with Visual Studio:
In the steps below I am going to integrate git plug in into Visual Studio 2010 Enterprise Edition.
Step 1:  Click on Tools menu and select Extension Manager.
Tools Menu
Step 2: In the extension manager select “Git Source Control Provider”. If not found you can go to Online Gallery tab search for git. You will find the listing as shown below. Click on the Download button against Git Source Control Provider.
Finding Git plugin

Step 3: It will start downloading the plugin. Upon download an installation dialog box popup. Select install. It will start the installation process.
Install Dialog Box
 Step 4: Upon successful installation of the plug in, Extension Manager will ask to restart the IDE. Click on “Restart Now” button.  Wait patiently until the IDE restarts.
Restart IDE message
Step 5: After the IDE has restarted, create any project and right click on Solution Explorer there you will see section for Git.
Git in action
