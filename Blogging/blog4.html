Regular Expressions:
The main function of regular expressions is search for a pattern within stream of strings. A regular expression gives the user the ability to search an expression with minimum efforts. There would be many times when we had to search for a pattern in a string and would have written lots of code to accomplish it. But it would have been better if some time is spend on learning regular expression and putting it in action.
How to use Regular Expressions in javascript?
To create a pattern in javascript, the general syntax would be
var variable-name = /pattern/;
Once the pattern is defined then, it could be used to search a particular pattern or more to replace a particular pattern.
Patterns Language:
Pattern
Description
\
Backslash is used to escape certain special symbols like ‘.’, ‘^’, ‘$’, ‘(‘, etc.
{n}
pattern needs to be matched exactly ‘n’ number of times
{n,}
pattern needs to match at least ‘n’ number of times
{n,m}
pattern needs to match at least ‘n’ and at most ‘m’ number of times
*
pattern could be matched ‘0’ or more number of times
+
pattern needs to match at least one time
?
pattern needs to match zero or more times
^
Search the pattern at beginning of string.
$
Search the pattern at the end of a string.
\b
Searches the pattern for a match at the word boundary i.e. either at the beginning or end of a word.
\B
Searches the pattern for a match excluding the word boundary.
\n
Searches for a New line character
\f
Searches for a form feed character
\r
Searches for a carriage return character
\t
Searches for a horizontal tab
\v
Searches for a vertical tab
[xyz]
Search for any character enclosed within the brackets. In this example pattern will be true, if any characters x, y or z is present.
[^xyz]
Will not search for characters not enclosed within.
\w
Matches for alphanumberic characters including underscores
\W
Matched for non alphanumberic characters including underscores
\d
Matches for any single digit.
\D
Matches for any non digit
\s
Matches for any space characters
\S
Matches any non-space characters
i
Ignores case of characters
G
Search for the all matches within the string
gi
Searches for all matches ignoring case

Email Validation Example:
There are several patterns available for email validation. However, the simplest technique is discussed over here.
For an email address of the form abc_xyz@def.com the pattern would be as follows:
\w+@\w+\.(com)
Pattern breakdown:
\w+     : used to recognize alphanumeric characters including underscore preceding ‘@’
@         : @ character is used to match for the presence of @ in between alphanumeric characters
\w+     : used to recognize alphanumeric characters after the @ symbol
\.       : used to identify the period character
(com) : used to identify the characters (com)
